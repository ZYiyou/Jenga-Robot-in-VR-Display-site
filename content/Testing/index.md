# Testing, Teamwork, and Reflection

## Testing
- Scenario-based integration tests to validate gameplay flow.
- Subsystem boundary tests ensuring modular functionality.
- Refactoring focused on maintainability and extensibility.

## Teamwork
- Coordinated team ownership across gameplay logic, robot execution, tower mechanics, voice commands, and presentations.
- Regular communication ensured integration clarity.

## Lessons Learned
- Clear integration boundaries accelerate debugging.
- Distributed ownership improves code quality.
- Strict scope control is vital for MVP success.
- Demo readiness influences development priorities.
